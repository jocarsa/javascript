Las colecciones de datos pueden tener múltiples dimensiones. La colección de datos que hemos visto en el ejercicio anterior, tiene únicamente una dimensión, por lo cual, los elementos presentan como si estuvieran tendidos a lo largo de una línea. En el siguiente ejemplo, vamos a mostrar el desarrollo de una estructura de datos de dos dimensiones. Para ello, en primer lugar, crearemos una colección de datos correspondiente a una agenda. Ahora, a continuación, en el primer registro, dentro de esa colección de datos, en lugar de asignarle directamente un valor, le asignaremos, a su vez, una colección de datos. Esto quiere decir que ahora, tenemos una colección dentro de otra colección. Y ahora así, en las siguientes líneas, es cuando podemos introducir datos en la segunda dimensión de la colección. Aprovecho este ejercicio para mostrar que el JavaScript tenemos una libertad bastante grande para especificar los índices de nuestras colecciones. Si bien en el primer índice hemos elegido trabajar con un índice de tipo numérico, en la segunda dimensión vamos a utilizar un índice de tipo alfanumérico, es decir, en lugar de introducir un número, podemos introducir una cadena de caracteres, tal como por ejemplo el nombre, el teléfono, o el e-mail. Muy importante es que esta cadena de caracteres debe ir entre comillas. Este ejercicio no quiere decir que el primer elemento de la colección de Basser numérico y la segunda dimensión de Baselga numérica, sino que lo que se pretende mostrar es que se pueden combinar ambas dos, tanto como se quiera. Por lo tanto, para la agenda, vamos a crear tanto un nombre, como un teléfono, como un e-mail. Podemos repetir esta operación tantas veces como haga falta, y por lo tanto, en el segundo elemento de la agenda, introduciremos, igualmente, una nueva colección de datos. Y, a continuación, especificaremos cada uno de los elementos que se encuentran dentro de esa segunda colección de datos, trabajando con los índices, alfanumérícoos. Por último, vamos a ejecutar una instrucción de salida por pantalla, en la cual podremos comprobar que el programa está guardando, en su memoria, el contenido completo de toda la colección de datos.
