hasta ahora hemos estado trabajando con estructuras de datos sencillas, natales como variables y constantes, que nos permiten introducir una pieza de información dentro de cada identificador. Sin embargo, si quiero hacer algo como por ejemplo una agenda, me daré cuenta de que las variables no son buenas estructuras para guardar múltiples piezas de información, ya que, en el caso de una agenda, sí escribo un registro, y luego sobre escribo otro registro, el primer registro se pierde, únicamente me quedo con el segundo, tal y como podemos ver en el ejemplo que se muestra a continuación. En primer lugar, declaro una agenda, y le asignó un valor. A continuación, cambio el valor introduciendo un nuevo nombre. Se a continuación saco el resultado por la pantalla, comprobaré que el primer valor a desaparecido, únicamente queda el segundo valor.
